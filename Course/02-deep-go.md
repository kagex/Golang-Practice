# Модуль 2. Глубокий Go 🟡

**Срок:** 4–6 недель  
**Цель модуля:** нырнуть под капот Go — интерфейсы, память, GC, дженерики. Этот модуль отличает девелопера от туториал-солдата.

---

## Урок 2.1. Интерфейсы

**📚 Теория:** неявная реализация, пустой интерфейс `interface{}`/`any`, type assertion `v, ok := x.(T)`, type switch.

**💻 Практика:** напиши функцию `Describe(any)`, которая выводит тип и значение через type switch.

**✅ Чек:** почему `var i interface{} = nil; i == nil` может быть `false` (nil-interface gotcha).

---

## Урок 2.2. Дизайн интерфейсов

**📚 Теория:** «маленькие интерфейсы — лучше», `io.Reader`/`io.Writer` как эталон, правило «accept interfaces, return structs», интерфейс на стороне потребителя.

**💻 Практика:** перепиши CLI Todo из Модуля 1 — введи интерфейс `TaskStore` с реализациями `JSONStore` и `InMemoryStore`.

---

## Урок 2.3. Композиция вместо наследования

**📚 Теория:** эмбеддинг структур и интерфейсов, повышение методов, диамонд-проблема и почему её нет в Go.

**💻 Практика:** `LoggingTaskStore` через эмбеддинг `TaskStore` + логирование перед каждым вызовом.

---

## Урок 2.4. Пакеты

**📚 Теория:** видимость через регистр (Exported/unexported), организация проекта (`cmd/`, `internal/`, `pkg/`), `internal`-пакеты, циклические импорты и как их ломать.

**✅ Чек:** разбей CLI Todo на `internal/store`, `internal/task`, `cmd/todo`.

---

## Урок 2.5. Generics (Go 1.18+)

**📚 Теория:** типовые параметры, constraints (`any`, `comparable`, `constraints.Ordered`), типовые set'ы, когда дженерики уместны (контейнеры, алгоритмы), когда нет (бизнес-логика).

**💻 Практика:** реализуй `Stack[T]`, `Queue[T]`, `Set[T]`, `Map[K comparable, V any]`.

---

## Урок 2.6. Reflection

**📚 Теория:** `reflect.Type`, `reflect.Value`, `Kind`, чтение тегов структур, цена рефлексии и когда оправдана (JSON-декодер, ORM).

**💻 Практика:** функция `PrintStruct(any)`, которая выводит все поля и их значения через рефлексию.

**⚠️ Красный флаг:** используешь reflection в бизнес-логике.

---

## Урок 2.7. Память: стек vs куча

**📚 Теория:** размещение объектов, escape analysis, правила escape (возврат указателя, замыкания, interface…), `go build -gcflags="-m"`.

**💻 Практика:** напиши 5 небольших функций, посмотри их escape analysis и объясни, почему так.

---

## Урок 2.8. Garbage Collector

**📚 Теория:** tri-color marking, write barriers, STW-фазы, `GOGC`, `GOMEMLIMIT`, влияние на latency, runtime.GC().

**✅ Чек:** объясни, что такое STW и за счёт чего Go GC делает его коротким.

---

## 🎯 Проект модуля

Библиотека-калькулятор с расширяемыми операциями через интерфейсы + набор дженерик-контейнеров (`Stack[T]`, `Queue[T]`, `Set[T]`) с покрытием тестами.

**Критерии приёмки:** добавление новой операции не требует правки существующего кода, дженерик-контейнеры покрыты тестами 80%+, escape analysis проверен.
