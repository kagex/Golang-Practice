# Модуль 3. Конкурентность 🟡

**Срок:** 3–4 недели  
**Цель модуля:** овладеть сердцем Go — горутинами, каналами и context. Без этого не возьмут даже на Junior+.

---

## Урок 3.1. Горутины

**📚 Теория:** запуск через `go func()`, цена создания (⁈2 КБ стек по умолчанию), отличия от потоков ОС, runtime.Gosched.

**💻 Практика:** запусти 100 одновременных горутин, посмотри память через `runtime.MemStats`.

---

## Урок 3.2. Планировщик GMP

**📚 Теория:** G (горутина), M (поток ОС), P (логический процессор), work-stealing, preemption (Go 1.14+).

**✅ Чек:** почему 1 млн горутин не убьёт процесс, а 1 млн потоков ОС убьёт.

---

## Урок 3.3. Каналы

**📚 Теория:** буферизованные vs небуферизованные, направление (`chan<-`, `<-chan`), `close`, `range`, send/recv на закрытый канал.

**💻 Практика:** пинг-понг между двумя горутинами через канал.

---

## Урок 3.4. select и таймауты

**📚 Теория:** мультиплексирование каналов, `default`, таймаут через `time.After`, отмена через `done`-канал.

**💻 Практика:** запрос к HTTP с таймаутом в 500 мс через `select`.

---

## Урок 3.5. Пакет `sync`

**📚 Теория:** `Mutex`, `RWMutex`, `WaitGroup`, `Once`, `Cond`, `Pool`. Читатели vs писатели, deadlock'и.

**💻 Практика:** потокобезопасный счётчик через `Mutex`, затем через `atomic` — сравни производительность.

---

## Урок 3.6. sync/atomic

**📚 Теория:** atomic-операции, `atomic.Int64`/`atomic.Pointer[T]`, compare-and-swap, когда лучше atomic, когда mutex.

---

## Урок 3.7. context.Context

**📚 Теория:** `context.Background`, `WithCancel`, `WithTimeout`, `WithDeadline`, `WithValue` (только для request-scoped!), распространение через стек вызовов.

**✅ Чек:** почему `context.Context` всегда первый аргумент функции.

**⚠️ Красный флаг:** хранишь в context user-объекты или пакеты данных.

---

## Урок 3.8. Паттерны конкурентности

**📚 Теория:** worker pool, fan-in / fan-out, pipeline, semaphore (через buffered channel), `errgroup.WithContext`, rate limiter.

**💻 Практика:** напиши worker pool на N воркеров, обрабатывающих задачи из канала, с graceful shutdown.

---

## Урок 3.9. Race Detector

**📚 Теория:** `go run -race`, `go test -race`, happens-before, data race vs race condition.

**💻 Практика:** напиши программу с data race намеренно, поправь с помощью детектора.

---

## 🎯 Проект модуля: Web Crawler

Многопоточный crawler:

принимает список URL
обходит их параллельно с лимитом (например 10 запросов одновременно)

корректно отменяется через context

собирает результаты без гонок

Критерии приёмки: `go test -race` чист, Ctrl+C корректно останавливает все горутины, лимит параллелизма работает.
