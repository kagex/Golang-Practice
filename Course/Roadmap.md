# 🗺️ Детальный Roadmap: Go Developer

## 🟢 Этап 1: Фундамент и идиоматичный Go (Недели 1–3)
*Цель: Научиться "думать на Go", а не писать на Go как на Python/Java.*
* **Модуль 00: Fundamentals** – Настройка окружения (Go 1.22+), `go mod`, структура workspace.
* **Модуль 01: Syntax** – Переменные, типы данных, управляющие конструкции, функции, методы.
* **Модуль 02: Deep Go** – Указатели, интерфейсы (duck typing), обработка ошибок (`errors.Is`, `errors.As`, `%w`), паника и восстановление (`panic`/`recover`).
* 🎯 **Практическая цель**: Написать CLI-утилиту (например, менеджер задач), которая читает JSON-файл, обрабатывает данные и выводит отформатированный отчет с корректной обработкой ошибок.
* ✅ **Чек-пойнт**: Вы понимаете разницу между значением и указателем, знаете, когда интерфейс должен быть значением, а когда указателем, и не используете `panic` для обработки бизнес-ошибок.

## 🟡 Этап 2: Конкурентность и Веб-разработка (Недели 4–6)
*Цель: Освоить главную фишку Go и научиться строить базовые HTTP-сервисы.*
* **Модуль 03: Concurrency** – Горутины, каналы (буферизированные/небуферизированные), `select`, `sync.WaitGroup`, `sync.Mutex`, паттерны (Worker Pool, Pipeline). *Изучить примеры из `03-concurrency-examples.md`.*
* **Модуль 04: Standard Library** – Глубокое погружение в `fmt`, `io`, `os`, `strings`, `time`, `encoding/json`.
* **Модуль 06: Web API** – Современный `http.ServeMux` (Go 1.22+ с поддержкой методов и `r.PathValue`), middleware, graceful shutdown, `http.Client` с таймаутами.
* 🎯 **Практическая цель**: Реализовать REST API для "Web Crawler" (из модуля 03), который принимает URL, сканирует его конкурентно (с ограничением через Worker Pool или `errgroup`) и отдает результат по HTTP с корректным таймаутом.
* ✅ **Чек-пойнт**: Ваш код не имеет data race (проверено через `go run -race`), все HTTP-клиенты имеют таймауты, а сервер корректно завершается по сигналу `SIGTERM`.

## 🟠 Этап 3: Данные, Тестирование и Производительность (Недели 7–9)
*Цель: Научиться писать надежный, быстрый и поддерживаемый код.*
* **Модуль 05: Databases** – `database/sql`, пулы соединений, транзакции, работа с PostgreSQL (pgx), миграции, N+1 проблема.
* **Модуль 07: Testing** – Table-driven tests, `testing`, моки (gomock/mockery), интеграционные тесты (testcontainers-go), fuzzing (Go 1.18+).
* **Модуль 11: Performance** – Профилирование (`pprof`: CPU, Memory, Block, Mutex), бенчмарки (`go test -bench`), Escape Analysis, оптимизация аллокаций.
* 🎯 **Практическая цель**: Добавить к вашему API базу данных (PostgreSQL), покрыть бизнес-логику unit-тестами (с моками) и интеграционными тестами. Написать бенчмарк и улучшить его показатели, используя `pprof`.
* ✅ **Чек-пойнт**: Покрытие тестами ключевых модулей > 80%, вы умеете читать вывод `go tool pprof` и находить источники лишних аллокаций в куче.

## 🔴 Этап 4: Архитектура и Микросервисы (Недели 10–12)
*Цель: Перейти от написания скриптов к проектированию промышленных систем.*
* **Модуль 10: Architecture** – Clean Architecture, Hexagonal Architecture, Dependency Injection, структура пакетов (Standard Go Project Layout).
* **Модуль 08: Microservices** – gRPC, Protobuf, межсервисная коммуникация, Message Brokers (Kafka, RabbitMQ), идемпотентность.
* **Модуль 15: Observability** – Логирование (`log/slog`), метрики (Prometheus, OpenTelemetry), распределенный трейсинг (Jaeger).
* 🎯 **Практическая цель**: Разбить монолитное API на два микросервиса (например, API Gateway и Worker), общающихся через gRPC. Добавить сквозное логирование с `trace_id` и базовые Prometheus-метрики.
* ✅ **Чек-пойнт**: Вы можете объяснить, почему выбрали Clean Architecture для конкретного проекта, и ваш сервис выдает структурированные логи с контекстом трассировки.

## 🟣 Этап 5: Продвинутые темы и Internals (Недели 13–15)
*Цель: Понимать, как Go работает "под капотом", для решения сложнейших задач.*
* **Модуль 13: Runtime Internals** – Планировщик GMP (Goroutine, Machine, Processor), Garbage Collector (tri-color mark and sweep, write barriers), работа с памятью.
* **Модуль 14: Distributed Systems** – Консенсус (Raft), распределенные блокировки, идемпотентные операции, паттерны устойчивости (Circuit Breaker, Retry).
* **Модуль 16: Security** – OWASP Top 10 для Go, защита от SQL-инъекций, XSS, безопасная работа с секретами, валидация входных данных.
* 🎯 **Практическая цель**: Написать кастомный аллокатор или оптимизировать критический участок кода, используя знания о GMP и GC. Реализовать Circuit Breaker для внешнего HTTP/gRPC вызова.
* ✅ **Чек-пойнт**: Вы можете нарисовать на доске схему работы планировщика Go и объяснить, что такое "preemption" в Go 1.14+.

## ⚫ Этап 6: Специализация и Карьера (Недели 16+)
*Цель: Адаптировать знания под конкретную нишу и подготовиться к собеседованиям.*
* **Модуль 09: DevOps** – Docker (multi-stage builds), CI/CD (GitHub Actions), линтеры (`golangci-lint`), Makefile.
* **Модуль 17: Cloud Native** – Написание Kubernetes Operators (controller-runtime), работа с cloud API.
* **Модуль 18: AI & Data** – Использование Go для высоконагруженных Data Pipeline, интеграция с ML-моделями (ONNX, cgo).
* **Модуль 19: Parsing** – Продвинутый парсинг, веб-скрейпинг, работа с `bufio` (обход `MaxScanTokenSize`), потоковая обработка больших данных.
* **Модуль 12: Interview** – Алгоритмы, системный дизайн, разбор типичных вопросов на Go (например, "как работает `defer` внутри?", "разница между `nil` слайсом и пустым слайсом").
* 🎯 **Практическая цель**: Собрать полноценное портфолио (3-4 законченных проекта с README, CI/CD и тестами), пройти 5-10 пробных собеседований по вопросам из Модуля 12.

---

### 💡 Как максимально эффективно использовать этот Roadmap:

1. **Не учите всё подряд**. Если вы целитесь в Backend, сделайте упор на Этапы 2, 3 и 4. Этапы 5 и 6 (Cloud Native, AI) можно изучать выборочно.
2. **Правило 20/80**: 20% времени на теорию (чтение модулей), 80% времени на практику (написание кода, рефакторинг, тесты).
3. **Используйте "Красные флаги"**: В каждом модуле оригинального репозитория есть раздел "Красный флаг" – это типичные ошибки. Держите их в голове при код-ревью своего кода.
