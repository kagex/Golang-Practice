# Область видимости в Go

## Введение

Область видимости (scope) определяет, в каких частях программы доступна та или иная переменная. В Go область видимости зависит от того, где была объявлена переменная.

## Основные принципы

### Блочная область видимости

В Go используются фигурные скобки `{}` для определения блоков. Переменные, объявленные внутри блока, видны только внутри этого блока.

```go
func example() {
    x := 10 // x видна только внутри функции example()
    
    if x > 5 {
        y := 20 // y видна только внутри блока if
        fmt.Println(x, y) // x и y доступны
    }
    
    // y здесь недоступна
    fmt.Println(x) // x доступна
}
```

### Область видимости переменных в if и switch

Переменные, объявленные в инициализаторе `if` или `switch`, видны во всем блоке, включая `else` и `case`.

```go
if x := 5; x > 0 {
    fmt.Println("x положительное:", x)
} else {
    fmt.Println("x не положительное:", x) // x доступна и в else
}

// x здесь недоступна
```

```go
switch x := 5; x {
case 5:
    fmt.Println("x равно 5:", x) // x доступна в case
default:
    fmt.Println("x не равно 5:", x) // x доступна и в default
}

// x здесь недоступна
```

#### Type Switch и область видимости

В type switch переменная, объявленная после `v.(type)`, доступна только внутри соответствующего `case`.

```go
var v interface{} = "привет"

switch val := v.(type) {
case int:
    fmt.Println("Это int:", val) // val доступна в этом case
case string:
    fmt.Println("Это string:", val) // val доступна в этом case
}

// val здесь недоступна
```

Каждый `case` имеет свою область видимости для переменной `val`.

### Переменные в циклах

В циклах `for` переменные, объявленные в инициализаторе, видны только внутри тела цикла.

```go
for i := 0; i < 5; i++ {
    fmt.Println(i) // i доступна внутри цикла
}

// i здесь недоступна
```

⚠️ **Важно для Go 1.22+**: Начиная с Go 1.22, каждая итерация цикла `for` создаёт новую переменную в инициализаторе. Это критически важно для конкурентного кода, особенно при использовании goroutines.

```go
// До Go 1.22: все goroutines использовали одну и ту же переменную i
// В Go 1.22+: каждая goroutine получает свою копию i
for i := 0; i < 5; i++ {
    go func() {
        fmt.Println(i) // В Go 1.22+ выводит правильное значение
    }()
}
```

Переменные, объявленные до цикла, доступны и внутри, и после него.

```go
i := 0
for i < 5 {
    fmt.Println(i)
    i++
}
fmt.Println("Итоговое значение i:", i) // i доступна
```

### Пакетная область видимости

Переменные, объявленные на уровне пакета (вне функций), видны во всех файлах этого пакета.

```go
// file1.go
package main

var packageVar = 100 // пакетная переменная

func func1() {
    fmt.Println(packageVar) // доступна
}
```

```go
// file2.go
package main

func func2() {
    fmt.Println(packageVar) // доступна из другого файла того же пакета
}
```

⚠️ **Важно**: Пакетная переменная не видна из других пакетов, даже если они импортируют этот пакет.

**Глобальные переменные** — это переменные, объявленные вне функций и видные во всем файле (если имя начинается с заглавной буквы — они экспортируются в другие пакеты).

**Локальные переменные** объявляются внутри функций или блоков и видны только внутри этих областей.

```go
func example() {
    localVar := 20 // локальная переменная
    fmt.Println(localVar)
}

// localVar здесь недоступна
```

## Перекрытие переменных

Можно объявить переменную с тем же именем, что и внешняя — это называется перекрытие (shadowing). Внутренняя переменная "закрывает" внешнюю.

```go
x := 10

if x > 5 {
    x := 20 // перекрытие переменной x
    fmt.Println(x) // 20 — используется внутренняя x
}

fmt.Println(x) // 10 — используется внешняя x
```

⚠️ **Важно**: Перекрытие может привести к путанице. Лучше использовать разные имена для избежания ошибок.

### Пример с вложенными блоками

```go
x := 10

if x > 5 {
    x := 20 // перекрытие внешней x
    fmt.Println(x) // 20
    
    if x > 15 {
        x := 30 // перекрытие x из предыдущего блока
        fmt.Println(x) // 30
    }
    
    fmt.Println(x) // 20 — снова внешняя x из первого блока
}

fmt.Println(x) // 10 — исходная переменная
```

## Область видимости функций

Функции, объявленные внутри пакета, видны только внутри этого пакета (если имя начинается с маленькой буквы).

```go
func privateFunc() { // видна только в пакете
    fmt.Println("Приватная функция")
}

func PublicFunc() { // видна из других пакетов
    fmt.Println("Публичная функция")
}
```

## Лучшие практики

1. **Минимизируйте область видимости**: Объявляйте переменные как можно ближе к месту использования. Это улучшает читаемость и снижает вероятность ошибок.

2. **Избегайте перекрытия**: Используйте разные имена для разных переменных, чтобы избежать путаницы.

3. **Используйте короткие имена для локальных переменных**: Например, `i` для счетчика цикла, `err` для ошибок.

4. **Глобальные переменные — последний resort**: Используйте их только когда действительно нужно. Лучше передавать данные через параметры функций.

## Область видимости в замыканиях

Замыкания (анонимные функции) захватывают переменные из внешней области видимости по ссылке, а не по значению.

```go
x := 10
f := func() {
    fmt.Println(x) // замыкание использует внешнюю x
}
x = 20
f() // выводит 20, а не 10
```

⚠️ **Важно**: Все замыкания, созданные в цикле, будут ссылаться на одну и ту же переменную, если не использовать дополнительные меры.

```go
// Проблема: все функции используют одну переменную i
var funcs []func()
for i := 0; i < 3; i++ {
    funcs = append(funcs, func() {
        fmt.Println(i)
    })
}
for _, f := range funcs {
    f() // выводит 3 три раза
}

// Решение: создать локальную копию
var funcs2 []func()
for i := 0; i < 3; i++ {
    j := i // локальная копия
    funcs2 = append(funcs2, func() {
        fmt.Println(j)
    })
}
for _, f := range funcs2 {
    f() // выводит 0, 1, 2
}
```

## Сравнение с другими языками

В отличие от Java или C#, где переменные видны внутри всего блока `if`, независимо от того, где они объявлены, в Go область видимости строго блочная.

```go
// Go - x НЕ видна после блока if
if true {
    x := 5
}
fmt.Println(x) // Ошибка: undefined: x
```

⚠️ **Важно**: В Java/C++ переменная, объявленная в блоке `if`, будет видна после него (если только не происходит выхода из блока). В Go — нет.

### Сравнение type switch

В type switch переменная после `.(type)` видна только внутри соответствующего `case`, но не в `default`:

```go
var v interface{} = "привет"

switch val := v.(type) {
case int:
    fmt.Println("Это int:", val)
default:
    fmt.Println("Не int") // val здесь недоступна
}

// val здесь недоступна
```

### Сравнение select

В `select` переменные, объявленные в `case`, видны только внутри этого `case`:

```go
ch1 := make(chan string)
ch2 := make(chan string)

select {
case msg := <-ch1:
    fmt.Println(msg) // msg видна
case msg := <-ch2:
    fmt.Println(msg) // msg видна
}
// msg здесь недоступна
```

Каждый `case` имеет свою область видимости для объявленных переменных.

## Заключение

Правильное понимание области видимости в Go помогает писать более чистый и безопасный код. Помните: чем меньше область видимости переменной, тем проще отладить программу.
