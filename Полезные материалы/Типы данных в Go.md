# Типы данных в Go

## Встроенные типы и алиасы

### Встроенные алиасы
- `byte` = `uint8`
- `rune` = `int32`
- `any` = `interface{}` (рекомендуется использовать `any`)

**Примечание:** `int` и `uint` — платформозависимы: 32 бита на 32-битных системах, 64 бита на 64-битных.

## Примитивные типы

### Числовые типы
- `bool` — логическое значение (true/false)
- `int`, `int8`, `int16`, `int32`, `int64` — целые числа (со знаком)
- `uint`, `uint8`, `uint16`, `uint32`, `uint64` — целые числа (без знака)
- `float32`, `float64` — числа с плавающей точкой
- `complex64`, `complex128` — комплексные числа

### Строковый тип
- `string` — неизменяемая последовательность байтов (кодировка UTF-8)

## Составные типы

### Массивы и срезы
- `array` — фиксированный размер, например: `[5]int`, `[10]string`
- `slice` — динамический массив, например: `[]int`, `[]string`

### Ассоциативные массивы
- `map` — словарь, например: `map[string]int`, `map[int]string`

### Структуры
- `struct` — пользовательский тип, например:
  ```go
  type Person struct {
      Name string
      Age  int
  }
  ```

**Именованные vs неименованные типы:**
- **Именованные** — объявлены через `type`, имеют имя (например, `type Person struct{...}`)
- **Неименованные** — строительные блоки без имени (например, `struct{...}`, `[]int`, `map[string]int`)
- Методы можно добавлять только к именованным типам

### Специальные типы
- `interface` — набор методов, например: `any`, `io.Reader`
- `chan` — канал для goroutines, например: `chan int`, `chan<- string`
- `pointer` — указатель, например: `*int`, `*Person`
- `function` — тип функции, например: `func(int) string`

### Сравнение типов (==, !=)

**Сравнимые типы (между собой и с nil):**
- `bool`, числовые типы
- `string`
- `pointer`
- `channel`
- `array` (если элементы сравнимы)
- `struct` (если все поля сравнимы)
- `interface` (см. примечание ниже)

**Несравнимые (можно сравнивать только с nil):**
- `slice`
- `map`
- `function`

**Важные нюансы сравнения:**
- `slice`, `map` и `function` нельзя сравнивать друг с другом (`map1 == map2` вызовет ошибку компилятора), их можно проверять только на инициализацию (`map1 == nil`)
- `interface` можно сравнивать, но если два интерфейса хранят значения несравнимых типов (например, оба содержат `slice`), сравнение вызовет `panic` во время выполнения программы
- `array` и `struct` можно сравнивать только в том случае, если все их внутренние элементы/поля тоже принадлежат к сравнимым типам

## Расширенные типы данных

### Указатели

Указатель хранит адрес памяти переменной и используется для:
- Передачи больших структур без копирования
- Изменения значения внутри функции
- Работы с адресами памяти

```go
var x int = 42
var p *int = &x
fmt.Println(*p) // 42
```

**Особенности:**
- Указатели позволяют избежать копирования больших структур
- Используются для изменения значения переменной внутри функции
- Нулевой указатель по умолчанию: `nil`

Срезы и карты уже содержат указатели внутри и передаются эффективно (копируется только заголовок).
См. подробный раздел: [Указатели (pointers).md](./Указатели%20(pointers).md)

### Интерфейсы
Интерфейс определяет набор методов. Любая реализация типа, который содержит все методы интерфейса, автоматически считается реализацией этого интерфейса.

```go
type Speaker interface {
    Speak() string
}

type Dog struct{}

func (d Dog) Speak() string {
    return "Woof!"
}

func MakeSound(s Speaker) {
    fmt.Println(s.Speak())
}
```

**Встроенные интерфейсы:**
- `error` — для обработки ошибок
- `io.Reader`, `io.Writer` — для операций ввода-вывода
- `comparable` — для типов, которые можно сравнивать

**Примечание:** `any` (алиас `interface{}`) используется как универсальный тип для значений любого типа.

### Функции как типы
Функции могут быть сохранены в переменные и передаваться как аргументы.

```go
type Calculator func(int, int) int

func add(a, b int) int {
    return a + b
}

func operate(a, b int, op Calculator) int {
    return op(a, b)
}

result := operate(5, 3, add)
```

### Каналы
Каналы используются для передачи данных между goroutines. Двунаправленные и односторонние каналы.

```go
ch := make(chan int)
go func() { ch <- 42 }() // Отправка
value := <-ch            // Получение

// Односторонние каналы
func sender(ch chan<- int) { ch <- 42 }
func receiver(ch <-chan int) { v := <-ch }
```

**Буферизованные каналы:**
```go
ch := make(chan int, 2)
ch <- 1
ch <- 2
// ch <- 3 // блокировка, буфер полон
```

## Дополнительные характеристики типов

### Размер типов
```go
import "unsafe"

fmt.Println(unsafe.Sizeof(int(0)))     // 8 байт на 64-битных системах
fmt.Println(unsafe.Sizeof(int8(0)))    // 1 байт
fmt.Println(unsafe.Sizeof(int16(0)))   // 2 байта
fmt.Println(unsafe.Sizeof(int32(0)))   // 4 байта
fmt.Println(unsafe.Sizeof(int64(0)))   // 8 байт
fmt.Println(unsafe.Sizeof(float32(0))) // 4 байта
fmt.Println(unsafe.Sizeof(float64(0))) // 8 байт
fmt.Println(unsafe.Sizeof(bool(false)))// 1 байт
```

### Пакет unsafe (использовать только когда действительно нужно)
- `unsafe.Pointer` — низкоуровневый указатель
- `unsafe.Sizeof`, `unsafe.Offsetof`, `unsafe.Alignof` — функции для низкоуровневой работы с памятью
- **Опасно**: нарушение памяти, неопределённое поведение

### Диапазоны значений
| Тип | Минимальное значение | Максимальное значение |
|-----|----------------------|------------------------|
| `int8` | -128 | 127 |
| `int16` | -32,768 | 32,767 |
| `int32` | -2,147,483,648 | 2,147,483,647 |
| `int64` | -9,223,372,036,854,775,808 | 9,223,372,036,854,775,807 |
| `uint8` | 0 | 255 |
| `uint16` | 0 | 65,535 |
| `uint32` | 0 | 4,294,967,295 |
| `uint64` | 0 | 18,446,744,073,709,551,615 |
| `float32` | ±1.4e-45 | ±3.4e+38 |
| `float64` | ±4.9e-324 | ±1.8e+308 |

### Нулевые значения
Каждый тип имеет значение по умолчанию, если не инициализирован:

| Тип | Нулевое значение |
|-----|------------------|
| `int`, `int8`, `int16`, `int32`, `int64` | `0` |
| `uint`, `uint8`, `uint16`, `uint32`, `uint64` | `0` |
| `float32`, `float64` | `0` |
| `bool` | `false` |
| `string` | `""` (пустая строка) |
| `pointer` | `nil` |
| `slice`, `map`, `chan` | `nil` |
| `interface` | `nil` |
| `function` | `nil` |
| `array` | обнуленные элементы (например, `[3]int` → `[0,0,0]`) |
| `struct` | обнуленные поля (например, `Person{Name:"", Age:0}`) |

### Время жизни и escape analysis
Go размещает переменную на стеке, если она используется только внутри функции и не выходит за её пределы. Если переменная выходит за пределы функции (например, её адрес передаётся), она размещается на куче. Это называется "escape analysis". Строки и массивы тоже могут "уехать" на кучу при возврате из функции.

## Преобразование типов

В Go явное преобразование обязательно:

```go
var i int = 42
var f float64 = float64(i)
var u uint = uint(f)

var s string = "Hello"
b := []byte(s) // string -> []byte
s2 := string(b) // []byte -> string
```

### Спецификация
- Все преобразования должны быть явными
- Потеря данных возможна при преобразовании между типами разного размера
- Преобразование между числовыми типами: `int` → `float64`, `int32` → `int64`
- Преобразование строки в байтовый срез и обратно: `string` ↔ `[]byte`

**Примечание:** `[]byte(s)` и `string(b)` создают копию данных, а не просто преобразуют тип.

**Важно:** прямое приведение `int("42")` или `string(123)` **не работает**. Используйте пакет `strconv`:
```go
import "strconv"

n, _ := strconv.Atoi("42")    // string → int
s := strconv.Itoa(42)         // int → string
```

## Методы и типы

Методы привязываются к именованным типам:

```go
type Person struct {
    Name string
    Age  int
}

func (p Person) Info() string {
    return fmt.Sprintf("%s is %d years old", p.Name, p.Age)
}

func (p *Person) Birthday() {
    p.Age++
}
```

**Примечания:**
- Метод с указателем (`*Person`) может изменять значение
- Метод со значением (`Person`) работает с копией
- Для изменения состояния используйте указатели

## Когда какой тип использовать

| Задача | Рекомендуемый тип |
|--------|------------------|
| Счётчик, индекс | `int` |
| Работа с Unicode | `rune` |
| Байтовые данные | `byte` / `[]byte` |
| Большие объёмы данных | `slice` |
| Быстрый поиск по ключу | `map` |
| Фиксированный размер | `array` |
| Передача больших структур | `*Struct` |
| Полиморфизм | `interface` |
| Конкурентность | `chan` |
| Хранение текста | `string` (неизменяемый) |
| Модификация данных | `[]byte` (изменяемый) |

## Рекомендации по использованию

1. **Используйте `int`** по умолчанию для целых чисел, если размер не критичен
2. **Предпочтение `slice`** над `array` для динамических коллекций
3. **Используйте `map`** только когда нужен быстрый поиск по ключу
4. **Избегайте избыточного использования указателей** — они усложняют код
5. **Используйте `rune`** для работы с отдельными Unicode-символами
6. **Используйте `byte`** для работы с байтовыми данными
7. **Понимайте семантику копирования** — `slice`, `map`, `chan` передаются как ссылки
8. **Используйте `uintptr`** только для низкоуровневых операций с памятью
9. **Проверяйте длину и ёмкость срезов** для избежания ошибок: `len()` и `cap()`
10. **Используйте интерфейсы** для абстракции и тестирования

## Примеры объявления переменных

```go
// Явное объявление
var age int = 25
var name string = "Ivan"

// Неявное объявление (инференс типов)
var isActive = true
var scores = []int{1, 2, 3}

// Краткая форма (внутри функций)
count := 10
message := "Hello"

// Map
person := map[string]any{
    "name": "Ivan",
    "age":  25,
}

type Point struct {
    X, Y int
}
p := Point{X: 10, Y: 20}

// Каналы
ch := make(chan int, 1)
ch <- 100
close(ch)

// Интерфейсы
var r io.Reader = os.Stdin
var e error = fmt.Errorf("error")
```
