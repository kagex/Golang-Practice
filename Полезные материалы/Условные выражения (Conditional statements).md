# Условные выражения в Go

## Введение

Операторы ветвления позволяют программе принимать решения на основе условий. Они определяют, какой блок кода будет выполнен в зависимости от истинности или ложности выражения.

## Основные операторы ветвления

### if

Базовый оператор условного перехода. Выполняет блок кода, если условие истинно.

#### Синтаксис

```go
if условие {
    // код, который выполнится, если условие истинно
}
```

#### Пример

```go
age := 20

if age >= 18 {
    fmt.Println("Пользователь совершеннолетний")
}
```

Если `age` больше или равно 18, программа выведет сообщение. Если нет — ничего не произойдет.

### if-else

Расширенная версия `if`, которая выполняет один блок кода, если условие истинно, и другой — если ложно.

#### Синтаксис

```go
if условие {
    // код, если условие истинно
} else {
    // код, если условие ложно
}
```

#### Пример

```go
age := 15

if age >= 18 {
    fmt.Println("Пользователь совершеннолетний")
} else {
    fmt.Println("Пользователь несовершеннолетний")
}
```

Если `age` больше или равно 18, выведется "Пользователь совершеннолетний". В противном случае — "Пользователь несовершеннолетний".

### if-else if-else

Позволяет проверить несколько условий последовательно. Выполняется только один первый блок, условие которого истинно.

#### Синтаксис

```go
if условие1 {
    // код, если условие1 истинно
} else if условие2 {
    // код, если условие2 истинно
} else if условие3 {
    // код, если условие3 истинно
} else {
    // код, если все условия ложны
}
```

#### Пример

```go
temperature := 30

if temperature < 0 {
    fmt.Println("Очень холодно!")
} else if temperature >= 0 && temperature <= 20 {
    fmt.Println("Прохладно")
} else if temperature > 20 && temperature <= 30 {
    fmt.Println("Тепло")
} else {
    fmt.Println("Жарко")
}
```

Проверки выполняются сверху вниз. Как только одно условие становится истинным, соответствующий блок выполняется, и остальные условия не проверяются.

### Оператор switch

Альтернатива цепочке `if-else if-else`. Позволяет выбрать один из многих блоков кода для выполнения на основе значения выражения.

#### Синтаксис basic switch

```go
switch выражение {
case значение1:
    // код, если выражение == значение1
case значение2:
    // код, если выражение == значение2
default:
    // код, если ни одно значение не совпало
}
```

#### Пример

```go
day := 3

switch day {
case 1:
    fmt.Println("Понедельник")
case 2:
    fmt.Println("Вторник")
case 3:
    fmt.Println("Среда")
case 4:
    fmt.Println("Четверг")
case 5:
    fmt.Println("Пятница")
case 6:
    fmt.Println("Суббота")
case 7:
    fmt.Println("Воскресенье")
default:
    fmt.Println("Некорректный номер дня")
}
```

Выведет "Среда", так как `day` равно 3.

#### Switch с несколькими значениями в одном case

```go
switch day {
case 1, 2, 3, 4, 5:
    fmt.Println("Рабочий день")
case 6, 7:
    fmt.Println("Выходной")
default:
    fmt.Println("Некорректный номер дня")
}
```

#### Switch без выражения (аналог if-else)

```go
temperature := 25

switch {
case temperature < 0:
    fmt.Println("Очень холодно")
case temperature >= 0 && temperature <= 20:
    fmt.Println("Прохладно")
case temperature > 20 && temperature <= 30:
    fmt.Println("Тепло")
default:
    fmt.Println("Жарко")
}
```

Этот вариант эквивалентен цепочке `if-else if-else`.

#### Switch с инициализацией (короткое объявление переменной)

В `switch` можно включить инициализацию переменной, которая будет доступна во всех `case`.

```go
score := 85

switch grade := score / 10; grade {
case 10, 9:
    fmt.Println("Отлично")
case 8:
    fmt.Println("Хорошо")
case 7:
    fmt.Println("Удовлетворительно")
case 6:
    fmt.Println("Плохо")
default:
    fmt.Println("Очень плохо")
}
```

Переменная `grade` объявляется в `switch`, и её область видимости ограничена всем блоком `switch`.

#### Fallthrough

Оператор `fallthrough` передаёт управление в следующий `case`, игнорируя его условие.

```go
number := 2

switch number {
case 1:
    fmt.Println("Один")
    fallthrough
case 2:
    fmt.Println("Два")
    fallthrough
case 3:
    fmt.Println("Три")
default:
    fmt.Println("Другое")
}
```

Вывод:
```
Два
Три
```

⚠️ **Важное ограничение**: `fallthrough` должен быть **последним оператором** в блоке `case`. Иначе компилятор выдаст ошибку.

```go
// Ошибка: fallthrough не может быть последним оператором в case
switch x := 2; x {
case 1:
    fmt.Println("Один")
case 2:
    fmt.Println("Два")
    fallthrough
    fmt.Println("Это не выполнится")
case 3:
    fmt.Println("Три")
}
```

✅ **Правило**: Если в `case` есть `fallthrough`, он должен быть последним, иначе — компиляция завершится с ошибкой.

### Оператор select

Используется для работы с каналами (goroutines). Выбирает одно из нескольких возможных операций обмена сообщениями (send/receive).

#### Синтаксис

```go
select {
case communication1:
    // код при получении/отправке через channel1
case communication2:
    // код при получении/отправке через channel2
default:
    // код, если ни один канал не готов
}
```

#### Пример

```go
ch1 := make(chan string)
ch2 := make(chan string)

go func() {
    time.Sleep(1 * time.Second)
    ch1 <- "сообщение из ch1"
}()

go func() {
    time.Sleep(2 * time.Second)
    ch2 <- "сообщение из ch2"
}()

select {
case msg1 := <-ch1:
    fmt.Println("Получено:", msg1)
case msg2 := <-ch2:
    fmt.Println("Получено:", msg2)
case <-time.After(500 * time.Millisecond):
    fmt.Println("Таймаут")
}
```

⚠️ **Важно**: В циклах `for` использование `time.After` может привести к утечкам памяти. Для циклов лучше применять `time.NewTimer` с явным сбросом через `Reset()`.

#### Дополнительные возможности select

**1. Случайный выбор при готовности нескольких case**

Если несколько case готовы одновременно, `select` выбирает один случайно (с равной вероятностью).

```go
ch1 := make(chan string)
ch2 := make(chan string)

go func() {
    ch1 <- "сообщение 1"
}()
go func() {
    ch2 <- "сообщение 2"
}()

select {
case msg1 := <-ch1:
    fmt.Println("ch1:", msg1)
case msg2 := <-ch2:
    fmt.Println("ch2:", msg2)
}
```

В каждом запуске может быть выбрано любое из сообщений — поведение непредсказуемо.

**2. Nil-каналы для динамического отключения кейсов**

Канал со значением `nil` никогда не готов ни для отправки, ни для получения. Это используется для динамического включения/отключения обработчиков.

```go
ch1 := make(chan string)
ch2 := make(chan string)

var disabledCh chan string

go func() {
    ch1 <- "активное сообщение"
}()

select {
case msg := <-ch1:
    fmt.Println("ch1:", msg)
case msg := <-ch2:
    fmt.Println("ch2:", msg)
case msg := <-disabledCh:
    fmt.Println("disabledCh:", msg)
}
```

В этом примере `disabledCh` равен `nil`, поэтому его case никогда не выберется. Если позже присвоить ему канал, обработчик активируется.

## Правила и лучшие практики

1. **Скобки обязательны**: В Go фигурные скобки `{}` для блоков кода обязательны, даже для одиночных операторов.

   ```go
   // Ошибка: синтаксическая ошибка
   if x > 0
       fmt.Println("x > 0")
   
   // Правильно
   if x > 0 {
       fmt.Println("x > 0")
   }
   ```

2. **Переменные в условиях**: Можно объявлять переменные в `if` и `switch`.

   ```go
   if x := 5; x > 0 {
       fmt.Println("x положительное")
   }
   ```

   Переменная `x` доступна только внутри блока `if`.

3. **Используйте switch вместо длинных цепочек if-else**: `switch` более читаем и эффективен для множественных условий.

4. **default в switch**: Всегда добавляйте `default`, чтобы обрабатывать неожиданные значения.

5. **Fallthrough редко нужен**: Используйте `fallthrough` только если вы действительно хотите выполнить следующий `case` без проверки его условия. Обратите внимание: `fallthrough` должен быть **последним оператором** в блоке `case`.

6. **Короткое замыкание (short-circuit evaluation)**: Логические операторы `&&` и `||` работают по принципу короткого замыкания — правое выражение вычисляется только если необходимо.

   ```go
   if p != nil && p.Value > 10 {
       fmt.Println("Значение больше 10")
   }
   ```

   Если `p` равен `nil`, правая часть `p.Value > 10` не выполняется, что предотвращает панику (nil pointer dereference).

   Аналогично для `||`:

   ```go
   if data != "" || cache != nil {
       // Данные есть ИЛИ есть кэш — использовать любой доступный источник
   }
   ```

7. **Type Switch (проверка типа в switch)**: Часто используется для определения типа переменной с интерфейсным типом.

   ```go
   var v interface{} = "привет"

   switch val := v.(type) {
   case int:
       fmt.Println("Это int:", val*2)
   case string:
       fmt.Println("Это string:", len(val))
   case []int:
       fmt.Println("Это slice int:", len(val))
   default:
       fmt.Println("Неизвестный тип")
   }
   ```

   Здесь `v.(type)` — специальный синтаксис type switch. Переменная `val` автоматически приводится к типу из `case`.

8. **Select и утечки ресурсов**: При использовании `time.After` в `select` в циклах возможна утечка памяти, так как таймеры не будут собраны сборщиком мусора до истечения времени.

   ```go
   // Потенциальная проблема в циклах
   for {
       select {
       case msg := <-ch:
           fmt.Println(msg)
       case <-time.After(1 * time.Second):
           fmt.Println("Таймаут")
       }
   }
   ```

   Для циклов лучше использовать `time.NewTimer` с явным сбросом:

   ```go
   timer := time.NewTimer(1 * time.Second)
   for {
       select {
       case msg := <-ch:
           fmt.Println(msg)
       case <-timer.C:
           fmt.Println("Таймаут")
       }
       timer.Reset(1 * time.Second)
   }
   ```

9. **Обработка ошибок в if (Go-паттерн)**: Классический паттерн обработки ошибок в Go.

   ```go
   if err != nil {
       return err
   }
   ```

   Это самая частая конструкция в Go-коде, используемая для проверки и немедленного возврата ошибки.

## Сравнение if и switch

| Критерий | if | switch |
|----------|----|--------|
| Количество условий | 1-2 | Множество |
| Читаемость | Снижается с ростом условий | Высокая даже при многих условиях |
| Производительность | Проверка по порядку | Оптимизация (jump table для плотных диапазонов) |
| Использование | Простые логические проверки | Сравнение значений выражения |

**Примечание**: Компилятор Go оптимизирует `switch` в jump table **только для плотных диапазонов целых чисел**. Для строк или разреженных значений используется последовательное сравнение, как в `if-else`.

## Заключение

Операторы ветвления — фундаментальная часть управления потоком выполнения в Go. Использование `if`, `else if`, `else` и `switch` позволяет создавать гибкие и понятные программы, способные реагировать на различные условия и входные данные.
