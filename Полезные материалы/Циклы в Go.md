# Циклы в Go

## Введение
Go следует философии минимализма и простоты. В языке существует **только одно ключевое слово для организации циклов** — `for`. 

Здесь нет `while`, `do-while` или `foreach`. Этот осознанный выбор упрощает чтение кода: разработчику не нужно гадать, какой именно тип цикла используется, достаточно увидеть `for` и проанализировать его условия. При этом один `for` гибко покрывает 100% сценариев итерации.

---

## 1. Три формы цикла `for`

### Классический цикл (C-style)
Используется, когда нужен точный контроль над индексом или шагом итерации.
```go
for i := 0; i < 5; i++ {
    fmt.Println(i) // 0, 1, 2, 3, 4
}
```
* `i := 0` — инициализация (выполняется один раз).
* `i < 5` — условие (проверяется перед каждой итерацией).
* `i++` — пост-действие (выполняется в конце каждой итерации).

### Цикл-условие (аналог `while`)
Если опустить инициализацию и пост-действие, `for` превращается в аналог `while` из других языков.
```go
count := 0
for count < 5 {
    fmt.Println(count)
    count++
}
```

### Бесконечный цикл
Если опустить все три части, цикл становится бесконечным. Это идиоматичный способ создания серверов, воркеров или игровых циклов.
```go
for {
    // Бесконечная работа
    // Выход осуществляется через break, return или panic
}
```

## 2. Возможности оператора `range`

`range` — это специальный оператор для итерации по коллекциям (слайсам, массивам, картам, строкам, каналам) и целым числам. Он делает код чище и безопаснее, но имеет важные нюансы.

⚠️ **Важно:** `range` не работает с произвольными структурами (`struct`). Компилятор выдаст ошибку.

### Итерация по слайсам и массивам
`range` возвращает **два** значения: индекс и **копию** значения элемента.
```go
nums := []int{10, 20, 30}

for i, v := range nums {
    fmt.Printf("index: %d, value: %d\n", i, v)
}
```
⚠️ **Критически важно:** `v` — это **копия** элемента. Изменение `v` внутри цикла **не изменит** исходный слайс.
```go
for _, v := range nums {
    v = 100 // ❌ Это меняет только локальную копию v, слайс nums остаётся прежним
}

// ✅ Правильный способ изменения:
for i := range nums {
    nums[i] = 100 // Меняем по индексу
}
```

💡 **Особенность массивов:** При итерации по массиву (`[N]T`, а не `[]T`) `range` работает с **копией всего массива**. Для больших массивов это может быть очень дорого. Поэтому для массивов всегда предпочтительнее использовать индекс (`for i := range arr`) или передавать указатель на массив.

### Итерация по целым числам (Go 1.22+)
Начиная с Go 1.22, `range` можно использовать с целыми числами. Это элегантная замена классическому циклу, когда нужен только счётчик.
```go
// Эквивалент for i := 0; i < 5; i++
for i := range 5 {
    fmt.Println(i) // 0, 1, 2, 3, 4
}

// Если индекс не нужен, можно использовать пустой идентификатор
for range 3 {
    fmt.Println("Hello") // Выведет 3 раза
}
```
*Переменная цикла не создаётся, так как количество итераций известно, а индекс не используется.*

### Итерация по строкам (String)
При итерации по строке `range` автоматически декодирует UTF-8 и возвращает **индекс байта** и **руну** (`rune` / `int32`), а не отдельные байты.
```go
s := "Привет"
for i, ch := range s {
    fmt.Printf("Байт: %d, Руна: %c\n", i, ch)
}
// Байт: 0, Руна: П
// Байт: 2, Руна: р (кириллица занимает 2 байта, поэтому индекс скачет)
```
*Если вам нужны именно байты, используйте классический `for i := 0; i < len(s); i++`. Если нужно узнать количество символов (рун) без итерации, используйте `utf8.RuneCountInString(s)`.*

### Итерация по картам (Map)
`range` по карте возвращает ключ и значение. 
⚠️ **Главное правило:** Порядок итерации по карте в Go **намеренно не определён и может отличаться между запусками программы**. Никогда не полагайтесь на порядок обхода `map`. Это позволяет разработчикам не полагаться на внутреннее устройство карты и делает код более устойчивым к изменениям реализации.
```go
m := map[string]int{"a": 1, "b": 2, "c": 3}
for k, v := range m {
    fmt.Println(k, v) // Порядок будет разным при каждом запуске!
}
```
*Если нужен сортированный порядок, сначала извлеките ключи в слайс, отсортируйте его (`sort.Strings`), и затем итерируйтесь по слайсу ключей.*

### Итерация по каналам (Channel)
Цикл будет получать значения из канала до тех пор, пока канал не будет закрыт (`close(ch)`).
```go
ch := make(chan int, 3)
ch <- 1
ch <- 2
close(ch) // Обязательно закрыть, иначе цикл зависнет навсегда (deadlock)

for v := range ch {
    fmt.Println(v) // Выведет 1, затем 2, затем цикл корректно завершится
}
```
⚠️ **Внимание:** Итерация по `nil`-каналу (`var ch chan int`) заблокирует горутину навсегда (deadlock), так как чтение из `nil`-канала всегда блокируется.

### Безопасность `nil`
Итерация по `nil`-слайсу или `nil`-карте **не вызывает панику**. Цикл просто не выполнится ни разу.
```go
var s []int
var m map[string]int

for _, v := range s { fmt.Println(v) } // Безопасно, 0 итераций
for k := range m { fmt.Println(k) }    // Безопасно, 0 итераций
```

## 3. Управление потоком: `break`, `continue` и метки (Labels)

### Базовое управление
* `continue` — пропускает остаток текущей итерации и переходит к проверке условия следующей.
* `break` — немедленно прерывает выполнение цикла.

```go
for i := 0; i < 10; i++ {
    if i%2 == 0 {
        continue // Пропускаем чётные числа
    }
    if i == 7 {
        break // Прерываем цикл на 7
    }
    fmt.Println(i) // Выведет: 1, 3, 5
}
```

### Метки циклов (Labeled Loops) и `switch`
В Go метки позволяют прерывать или продолжать **внешний** цикл из вложенного. Это особенно важно, так как `break` внутри блока `switch` или `select` прерывает только этот блок, а не окружающий цикл.

```go
OuterLoop:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i == 1 && j == 1 {
            break OuterLoop // Прерывает ВЕСЬ внешний цикл, а не только внутренний или switch
        }
        fmt.Printf("i=%d, j=%d\n", i, j)
    }
}
```

## 4. Особенности работы цикла

### Семантика переменных цикла (Go 1.22+)
Это одна из самых важных тем, тесно связанная с замыканиями и горутинами.

**До Go 1.22:** Переменные цикла (`i` и `v` в `for i, v := range`) создавались **один раз** перед циклом. Каждая итерация перезаписывала их значения. Это приводило к знаменитой ошибке при захвате переменной в замыкание или горутину (все горутины видели последнее значение).

**Начиная с Go 1.22:** Компилятор создает **новую переменную** для каждой итерации цикла. 
```go
var funcs []func()
for i := range 3 {
    funcs = append(funcs, func() { fmt.Println(i) })
}
// В Go 1.22+ это корректно выведет: 0, 1, 2
// В Go < 1.22 это вывело бы: 3, 3, 3
```
💡 *Совет:* Несмотря на исправление в Go 1.22, при захвате **указателей** (`&v`) в цикле всё равно рекомендуется создавать явную локальную копию или брать адрес из исходного слайса (`&slice[i]`), чтобы избежать путаницы при чтении кода.

## 5. Подводные камни

### 1. ⚠️ Изменение слайса во время итерации
Добавление элементов в слайс (`append`) внутри цикла `range` может привести к непредсказуемому поведению. `range` вычисляет длину слайса **один раз** в начале итерации. Если произойдет реаллокация (изменение `cap` при `append`), `range` продолжит итерацию по старой памяти, что может привести к панике (out of bounds) или пропуску элементов.

```go
// ❌ Опасно: может привести к панике или пропуску элементов
nums := []int{1, 2, 3}
for _, n := range nums {
    if n == 2 {
        nums = append(nums, 99) 
    }
}

// ✅ Безопасно: создаем новый слайс или итерируемся по копии длины
var result []int
for _, n := range nums {
    result = append(result, n)
    if n == 2 {
        result = append(result, 99)
    }
}
```

### 2. ⚠️ Бесконечный цикл из-за `uint`
Использование беззнаковых целых чисел (`uint`) для обратного отсчета до нуля — классическая ловушка. При вычитании из `0` происходит переполнение до максимального значения.
```go
// ❌ Бесконечный цикл!
for i := uint(3); i >= 0; i-- {
    fmt.Println(i) // 3, 2, 1, 0, 18446744073709551615, ...
}

// ✅ Правильно: используем знаковый int
for i := 3; i >= 0; i-- {
    fmt.Println(i)
}
```

## 6. Производительность и лучшие практики

### Мифы о производительности: `len()` и `range`
**Что быстрее: `for i := 0; i < len(s); i++` или `for i := range s`?**
В современных версиях Go (1.22+) `for i := range s` часто работает быстрее или так же, потому что компилятор **может** избежать копирования значения элемента, если используется только индекс, и генерирует более оптимальный код. Всегда проверяйте с помощью `go test -bench`.

**Нужно ли выносить `len()` в переменную?**
Хотя вызов `len()` для слайсов, строк и массивов имеет константную сложность O(1) и обычно оптимизируется компилятором (поэтому выносить его в переменную не обязательно с точки зрения производительности), если условие цикла содержит более сложные вызовы функций, их лучше вычислить заранее для улучшения читаемости.
```go
// ✅ Хорошо: вычисляем один раз для читаемости
n := len(data)
for i := 0; i < n; i++ { ... }
```

### Pre-allocation (Предварительное выделение памяти)
Если вы знаете итоговый размер слайса, создавайте его с нужной емкостью (`cap`), чтобы избежать дорогостоящих реаллокаций внутри цикла.
```go
// ❌ Медленно: множество реаллокаций
var res []int
for i := 0; i < 1000; i++ { res = append(res, i) }

// ✅ Быстро: одна аллокация
res := make([]int, 0, 1000)
for i := 0; i < 1000; i++ { res = append(res, i) }
```

### Игнорируйте ненужные значения через `_`
Это не только удовлетворяет компилятор, но и явно сигнализирует читателю кода о ваших намерениях, а в Go 1.22+ предотвращает лишнее копирование данных.
```go
for _, val := range data { ... } // Нам важен только val
for i := range data { ... }      // Нам важен только индекс
```

## 7. FAQ

**Как перебрать слайс в обратном порядке?**
Используйте классический цикл `for`, `range` не поддерживает обратный обход.
```go
for i := len(slice) - 1; i >= 0; i-- {
    fmt.Println(slice[i])
}
```

**Почему `range` по строке возвращает `rune`, а не `byte`?**
Потому что строки в Go — это неизменяемые срезы байтов, но они часто содержат UTF-8 текст. `range` автоматически выполняет декодирование UTF-8, что избавляет разработчика от ручного написания сложных циклов с битовыми сдвигами для обработки многобайтовых символов (например, эмодзи или кириллицы).

**Можно ли использовать `goto` для выхода из цикла?**
Технически да, но это считается плохой практикой (anti-pattern), так как усложняет чтение кода. Используйте именованные метки (`break Label`) — это идиоматичный и безопасный способ Go.

## 8. Краткое резюме

1. В Go есть только один оператор цикла: `for`. Он универсален.
2. `range` — лучший инструмент для итерации по коллекциям, но помните: он возвращает **копию** значения, а не ссылку на него.
3. Порядок итерации по `map` намеренно не определён. Никогда не полагайтесь на него.
4. В Go 1.22+ семантика переменных цикла изменена: каждая итерация получает свою собственную переменную, что решает старые проблемы с замыканиями. Также добавлена итерация по целым числам (`for i := range 10`).
5. Для изменения элементов слайса используйте индекс (`slice[i] = newValue`), а не переменную из `range`.
6. Итерация по `nil` слайсу или карте безопасна и просто не выполняет тело цикла, но итерация по `nil`-каналу вызовет deadlock.
7. Используйте `continue` для пропуска итераций и именованные метки (`break Label`) для выхода из вложенных циклов.
8. Предварительно выделяйте память (`make` с `cap`) для слайсов, которые наполняются в цикле.
